핸드폰에 IP와 포트를 입력하는 것은 클라이언트 애플리케이션에서만 필요합니다. 서버는 일반적으로 고정된 IP 주소를 가지고 있으며, 서버 애플리케이션을 실행할 때 포트 번호를 설정하여 클라이언트의 연결을 기다립니다.

구체적인 설명
클라이언트 애플리케이션 (핸드폰)

사용자는 핸드폰 애플리케이션에서 서버의 IP 주소와 포트 번호를 입력합니다.
이 정보는 클라이언트가 서버에 연결할 때 사용됩니다. 클라이언트는 입력된 IP 주소와 포트 번호로 서버에 접속을 시도합니다.
서버 애플리케이션

서버 애플리케이션은 서버의 IP 주소를 고정된 값으로 가지고 있습니다. 예를 들어, 서버가 실행되는 컴퓨터의 네트워크 IP 주소를 사용합니다.
서버는 특정 포트 번호를 지정하여 클라이언트의 연결을 기다립니다. 클라이언트가 이 포트 번호로 연결을 시도하면 서버는 해당 포트에서 클라이언트의 접속을 수락합니다.
예시
클라이언트 애플리케이션 (핸드폰)

IP 입력란에는 서버의 IP 주소를 입력합니다. 예를 들어, "192.168.0.1"과 같은 형태입니다.
Port 입력란에는 서버에서 지정한 포트 번호를 입력합니다. 예를 들어, "7777"과 같은 포트 번호입니다.
서버 애플리케이션

서버는 IP 주소 설정이 필요 없으며, 실행되는 컴퓨터의 네트워크 환경에 따라 자동으로 IP 주소를 할당받습니다.
포트 번호는 서버 애플리케이션을 실행할 때 설정하며, 클라이언트가 이 포트 번호로 접속할 수 있도록 대기합니다.
결론
클라이언트에서는 서버의 IP 주소와 포트 번호를 입력하여 서버에 접속하고, 서버는 포트 번호를 설정하여 클라이언트의 접속을 기다립니다. 서버에서는 일반적으로 IP 설정은 필요하지 않습니다.
